# Markdown 标题

# 基本语法

使用 # 号标记
Markdown 使用 # 号来创建标题，这是从 HTML 的 <h1> 到 <h6> 标签概念演化而来的。
使用 # 号可表示 1-6 级标题，一级标题对应一个 # 号，二级标题对应两个 # 号，以此类推。
标题的数量 # 号的数量不能超过 6 个，超过 6 个 # 号将被视为普通段落。

# 注意事项

符号与文字间的空格：# 号和标题文字之间必须有一个空格。这是标准的 Markdown 语法要求。

# Markdown 文本格式

使用 <!-- 注释内容 --> 来注释内容

Markdown 段落没有特殊的格式，直接编写文字就好，段落的换行是使用两个以上空格加上回车。

# 字体

文本强调是写作中的重要技巧，Markdown 提供了简洁的方式来实现粗体和斜体效果。

Markdown 可以使用以下几种字体：粗体和斜体。

粗体语法:​使用两个星号 ** 或两个下划线 __ 包围文字。
斜体语法:​使用一个星号 * 或一个下划线 _ 包围文字。
斜体和粗体:​使用三个星号 *** 或三个下划线 ___ 包围文字。

# 为什么 Markdown 加粗语法（……）有时候不生效？

…… 那猹却将身一扭，反从他的胯下逃走了。** 这少年便是闰土。** 我认识他时，也不过十多岁，离现在将有三十年了；

如上，Markdown 渲染失效了……

开始标记符的前后必须满足：

  1. 不能后跟空白
  2. 标点符号
  3. 不能后跟标点符号
  4. 如果后跟标点符号，则前面必须有空白

结束标记符的规则和开始标记符相反：

  1. 前面不能是空白
  2. 标点符号
  3. 前面不能是标点符号
  4. 如果前面是标点符号，则后面必须有空白

结合上方的例子，这里的 ** 的前面是标点符号，但后面却不是空白或标点符号，所以它不是右侧定界符序列，不会被当作是结束粗体的标识符，自然粗体就不生效了。

怎么解决？

  1. 可以使用 HTML 标签 <b> 或 <strong> 来实现粗体效果。
  2. 可以使用 HTML 标签 <i> 或 <em> 来实现斜体效果。
  3. 在 ** 后面加个空格即可。

# 列表嵌套

Markdown 列表可以嵌套使用，即列表项中可以包含子列表。
子列表的缩进是通过在列表项前添加空格或制表符来实现的。

    2. 子项 2
  2. 第二项
    • 子项 1
    • 子项 2

嵌套规则：

  1. 子列表的缩进是通过在列表项前添加空格或制表符来实现的。（推荐 2 个）
  2. 子列表的标记可以是星号 * 、加号 + 或减号 - ，但必须与父列表的标记相同。
  3. 子列表的数字标记可以省略，Markdown 会自动编号。

表格和引用是 Markdown 中重要的内容组织工具。
表格能够清晰地展示结构化数据，而引用则用于突出重要信息或引用他人观点。

语法要点：

  • 表头和数据行之间必须有分隔线
  • 分隔线至少需要三个连字符 ---
  • 两端的竖线 | 是可选的，但建议保留以提高可读性
  • 不需要严格对齐，但对齐后更美观

对齐方式：
我们可以设置表格的对齐方式：

  • ---: 设置内容和标题栏居右对齐。
  • :--- 设置内容和标题栏居左对齐。
